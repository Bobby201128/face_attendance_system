## 🎯 核心功能实现

### 1. 环境系统 🌍

- **独立配置**: 每个环境独立的工作时间和识别参数
- **灵活切换**: PC启动时选择环境，支持默认环境
- **技术实现**:

- **树形结构**: 支持无限层级嵌套
- **拖拽排序**: 灵活的分类排序
- **技术实现**:
  - 数据库表: `categories`
  - API端点: 7个
  - UI组件: 分类树管理界面

- **工作流优化**: 上传 → 待审核 → 已批准/已拒绝
- **技术实现**:

  - UI组件: 人脸审核对话框、上传界面

## 🛠️ 技术栈

### 后端技术
- **数据库**: SQLite (WAL模式，外键约束)
- **API框架**: Flask (RESTful设计)
- **人脸识别**: face_recognition + dlib
- **图像处理**: OpenCV + NumPy

### 前端技术
- **PC端**: PyQt5 (桌面应用)
- **移动端**: HTML5 + CSS3 + JavaScript
- **响应式设计**: 适配各种屏幕尺寸

### 开发工具
- **版本控制**: Git
- **环境管理**: Conda
- **自动化脚本**: Python

## 📱 使用指南

### 系统启动
```bash
# 1. 激活环境
conda activate face_attendance

# 2. 启动系统
python main.py

# 3. PC端会自动显示环境选择对话框
# 4. 移动端访问: http://<本机IP>:5000
```

### 基础操作
1. **创建环境**: 移动端 → 设置 → 环境管理
2. **建立分类**: 移动端 → 设置 → 分类管理
3. **上传人脸**: 移动端 → 人员 → 上传照片
4. **审核人脸**: PC端 → 人脸审核按钮
5. **分配环境**: PC端 → 人员环境按钮
